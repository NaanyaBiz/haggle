# Secure-SDLC conformance map

This repository operates under the committed
[secure-SDLC standard](./secure-sdlc-standard.md). This map is the
**operative, public, statement-level record** of how each of the standard's
94 control statements is met, the alternate control used, or the
recorded exception. It is a living document: any PR that changes a control
surface (workflow, ruleset baseline, agent grant, release gate, security
doc) updates the affected rows in the same PR.

## Self-assessment declaration (CO-1.1)

- **Criticality tier: Tier 3 (Standard).** Consequence basis: one
  household's energy data and one AGL OAuth session per install, on the
  user's own Home Assistant instance (SECURITY.md § Impact assessment —
  low-to-moderate direct impact; the high-impact factor is supply-chain
  reach across the installed base, which is why the supply-chain controls
  below are operated far above tier).
- **Authoring stratum: S1 (engineered)**, with the **S5 agentic overlay**
  (CO-12) applying to the committed agent artifacts (`.claude/**`,
  `docs/agents/**`).
- Most controls are deliberately operated at **Tier 1–2 intensity** despite
  the Tier 3 declaration: this repository doubles as a reference
  implementation of the standard. Where a row below notes a statement is
  above tier scope, the control is operated voluntarily.
- **Role functions collapse into the owner** (standard §1, Interpretation):
  the maintainer holds all first-line accountabilities personally; acceptances
  are owner-signed (SECURITY.md register — merge is the signature). Statements
  requiring a function independent of the owner are recorded exceptions, never
  reinterpretations.
- Re-validated on material change and at least annually (next: 2027-07).

## Method and the private review

A full statement-level conformance review (multi-agent, adversarially
verified) was performed in July 2026 and is repeated after material
remediation. The detailed review report and work papers are the
maintainer's proprietary material and are **not** published in this
repository. This map is the public summary of the operative position:
every mechanism cited below refers to committed artifacts or live GitHub
platform state and is independently re-verifiable without the private
material.

## Status legend

| Status | Meaning |
|---|---|
| **met** | The statement is enforced by the cited mechanism as written |
| **met-alternate** | The objective is met **in full** through an *alternate control* — a different mechanism than the statement's letter, at the stratum/scale expression the standard itself anticipates (Section 6). An alternate control is not a *compensating control*: a compensating control partially offsets a control that remains absent, appears only in the narrative of partial/accepted/structural rows, and never upgrades a status |
| **partial** | A real mechanism operates but does not cover the statement's full letter |
| **accepted** | Gap with a recorded, dated risk acceptance (SECURITY.md register RA-xx or cited paragraph) |
| **structural** | Single-maintainer impossibility (no second human); recorded in the register and revisited if a co-maintainer joins |
| **n/a** | The statement has no referent in this repository (reason given) |
